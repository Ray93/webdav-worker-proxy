# 管理台前端视觉重构设计

日期：2026-04-20

## 目标

在不改动现有功能流和交互逻辑的前提下，重构 Cloudflare Worker WebDAV 中转代理的管理台前端视觉。

本次范围仅包含：

- 初始化页视觉重构
- 登录页视觉重构
- 规则页视觉重构
- 路由弹窗视觉重构
- 统一颜色、字体、卡片、按钮、表单、状态标签样式

本次明确不包含：

- 不改 API
- 不改状态流
- 不改初始化逻辑
- 不改登录逻辑
- 不改路由编辑交互方式
- 不新增功能

## 总体方向

采用 `Swiss Workbench` 风格。

它的目标不是做成营销官网，也不是传统后台，而是做成一个偏专业、克制、理性的精致工作台。

视觉关键词：

- 浅底
- 深字
- 单强调色
- 清晰网格
- 强信息层级
- 少装饰
- 轻动效

## 整体视觉系统

### 背景

- 主背景使用冷白或浅灰白
- 去掉当前明显的彩色渐变和玻璃拟态主导风格
- 允许保留非常轻的背景层次，但不能抢内容

### 配色

- 主文字：深海军蓝或接近黑蓝
- 次文字：中灰蓝
- 页面背景：冷白、浅灰白
- 强调色：单一蓝色
- 成功状态：冷绿色，仅用于状态语义
- 危险状态：克制的红色，仅用于删除或错误提示

### 卡片

- 卡片采用浅底、细边框、轻阴影
- 层级主要通过留白、对齐、字体权重建立
- 不依赖大面积模糊、高透明、彩色反光

### 圆角

- 保留圆角，但比当前更克制
- 所有卡片、输入框、按钮使用统一圆角尺度

### 字体

- 标题和正文都采用专业无衬线字体体系
- 标题强调清晰和秩序，不做夸张展示化处理
- 技术信息块使用等宽字体，但仅限 Secret、URL、Header 等内容

### 动效

- 仅保留 hover、focus、弹窗进入等轻量过渡
- 过渡时长控制在 150ms 到 250ms
- 不使用持续性装饰动画
- 需要兼容 `prefers-reduced-motion`

## 页面视觉布局

### 初始化页

- 保留双栏结构
- 左侧为视觉引导区
- 右侧为操作面板

左侧内容：

- 小标签
- 大标题
- 一段简短说明
- 2 到 3 张信息卡片

右侧内容：

- 创建密码状态时显示表单卡片
- 配置 Secret 状态时显示 Secret 信息卡片与操作按钮

Secret 展示方式：

- 使用轻量代码信息块
- 强调可复制感
- 不做开发者文档风格

### 登录页

- 延续初始化页双栏结构
- 左侧承担说明和整体气质
- 右侧为登录卡片
- 页面切换时要保持同一套产品语言

### 规则页

- 顶部采用工作台页头
- 左侧：标题和说明
- 右侧：主要操作按钮

摘要区：

- 保留规则总数、已启用、保留原路径三张摘要卡
- 摘要卡偏扁平、克制，不做仪表盘感

规则列表：

- 继续单列卡片流
- 每张卡片按以下层级组织

1. 标题区：前缀 + 状态
2. 主信息区：目标地址
3. 辅助信息区：路径处理方式 + 请求头数量
4. 技术区：请求头 code block
5. 操作区：编辑 / 启用停用 / 删除

### 路由弹窗

- 保留居中弹窗
- 改成更轻量的编辑面板视觉
- 强化标题、描述、字段、操作区之间的节奏
- 请求头编辑列表更规整，但不改变交互模式

## 组件规范

### 标题系统

- 页面主标题明显放大，但不夸张
- 副标题控制在 1 到 2 行
- 小标签统一使用极简胶囊标签

### 按钮系统

- 主按钮使用实心深色或主蓝色
- 次按钮使用描边或浅底
- 危险按钮保留红色语义，但降低刺激性

### 表单系统

- 输入框边框更清楚
- 聚焦态更明确
- 内边距略大于当前版本
- 标签更紧凑
- 错误信息仅做局部提示

### 状态系统

- 启用 / 停用 / 保留前缀 / 不保留前缀 统一为轻量标签
- 颜色仅为辅助
- 文字本身必须完整表达语义

### 技术信息块

- Secret、目标地址、请求头统一使用轻量等宽信息块
- 增强可读性和复制感
- 避免形成终端风或开发文档风

## 响应式要求

- 桌面端保留双栏和清晰分区
- 移动端折叠为单列
- 按钮在小屏下更接近满宽
- 不允许出现横向滚动

## 成功标准

完成后应满足：

- 视觉明显比当前版本更克制、更专业
- 初始化页、登录页、规则页、弹窗看起来属于同一系统
- 功能流和交互逻辑保持不变
- 页面不再依赖彩色玻璃拟态来建立高级感
- 管理台更像精致工作台，而不是营销页或传统后台
